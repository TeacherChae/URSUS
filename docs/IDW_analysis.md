# IDW(Inverse Distance Weighting) 분석

> URSUS 프로젝트에서 사용하는 공간 보간 알고리즘의 원리·기능·실무 맥락·장단점 총정리

---

## 1. 서론: 왜 보간이 필요한가

우리가 다루는 도시 데이터는 본질적으로 **이산(discrete)** 하다. 서울시 행정 체계는 도시를 법정동·행정동 단위로 분할하고, 각 단위마다 하나의 대표값(소득, 인구, 승차 승객 수)을 부여한다. 결과적으로 우리가 가진 데이터는 수백 개의 점(법정동 중심점)과 그에 대응하는 스칼라값의 집합이다.

그런데 도시 공간에서 인간 활동이 실제로 법정동 경계에서 딱 잘라지진 않는다. 성수동과 왕십리동 사이 어딘가에도 사람이 살고 있고, 그 지점의 "소득 수준"이나 "이동 밀도"는 두 법정동의 값을 어떤 비율로 반영할 것이다. **보간(interpolation)** 은 바로 이 "사이 지점"의 값을 추정하는 행위다.

URSUS에서는 메쉬(Mesh)의 각 버텍스가 수천~수만 개의 공간 점을 이루고, 그 각각에 값을 부여해야 하므로 보간이 필수적이다.

---

## 2. IDW의 핵심 아이디어: 가까울수록 더 닮는다

IDW의 철학은 단순하다.

> **어떤 미지점의 값은, 그 점에 가까운 관측점일수록 더 강하게 반영된다.**

이 직관은 지리학의 제1법칙(Tobler's First Law of Geography)과 일치한다.

> *"Everything is related to everything else, but near things are more related than distant things."*
> — Waldo Tobler, 1970

IDW는 이 원칙을 수식으로 구현한 것이다.

---

## 3. 수식: 정확한 동작 원리

### 3.1 기본 수식

```
F(q) = Σᵢ [ vᵢ / dᵢᵖ ]  /  Σᵢ [ 1 / dᵢᵖ ]
```

| 기호 | 의미 |
|---|---|
| `F(q)` | 미지점 q에서의 추정값 |
| `vᵢ` | i번째 관측점의 알려진 값 |
| `dᵢ` | 미지점 q와 i번째 관측점 사이의 거리 |
| `p` | 거리 지수(power parameter) |

이 수식은 **가중 평균**이다. 분자는 각 관측값에 가중치(`1/dᵢᵖ`)를 곱한 합, 분모는 가중치의 합. 분모로 나누기 때문에 결과는 항상 관측값의 최솟값과 최댓값 사이에 존재한다는 것이 보장된다.

### 3.2 특수 케이스: q가 관측점과 완전히 일치할 때

`dᵢ = 0`이면 `1/dᵢᵖ`이 무한대가 되어 수식이 발산한다. 이 경우 해당 관측점의 값을 그대로 반환하면 된다. URSUS 구현에서는 `d2 < 1e-20` 조건으로 처리한다.

```csharp
if (d2 < 1e-20) return Values[i];  // SpatialField.cs:147
```

### 3.3 거리 지수 p의 역할

`p`는 IDW에서 가장 결정적인 파라미터다.

- `p`가 **클수록**: 가까운 점의 영향이 급격히 커지고, 먼 점의 영향은 무시된다. 결과는 보로노이 다이어그램에 가까워진다 — 각 점이 자신에게 가장 가까운 관측점의 값을 그대로 가져오는 형태.
- `p`가 **작을수록**: 먼 점도 유의미한 가중치를 가지므로 보간 면이 부드럽게 퍼진다.
- `p = 0`: 모든 점에 동일 가중치 → 단순 산술 평균 (거리 무관)
- `p = 2`: 가장 널리 쓰이는 기본값 (면적 비례 가중)
- `p = 3`: URSUS 기본값 — 상대적으로 국소적(local) 영향권

### 3.4 URSUS의 성능 최적화: 정수 지수 직접 곱셈

`Math.Pow(d, p)`는 내부적으로 `exp(p * log(d))`로 계산되어 부동소수점 트랜센덴털 함수 비용이 발생한다. URSUS에서는 p가 정수(1~6)인 경우를 감지해 반복 곱셈으로 처리한다.

```csharp
bool intPow = (ip >= 1 && ip <= 6 && Math.Abs(power - ip) < 1e-9);
if (intPow) {
    double d = Math.Sqrt(d2);
    double dp = d;
    for (int j = 1; j < ip; j++) dp *= d;
    w = 1.0 / dp;
}
```

p=3이라면 `d * d * d` 세 번의 곱으로 끝난다. 수십만 버텍스 × 수백 관측점 루프에서 누적 효과가 상당하다.

## 4. URSUS에서의 전체 파이프라인

IDW는 URSUS 시각화 파이프라인의 한 단계다. 전체 흐름을 맥락과 함께 짚어보자.

```
법정동 중심점 (Point3d)  ←  VWorld Geocoder API
법정동 오버레이 값 (0~1)  ←  BuildOverlayValues() [min-max 정규화 + 균등 평균]
         ↓
     IDWVisualizer
         ↓
  메쉬 버텍스 생성 (MeshingParameters)
         ↓
  버텍스마다 IDW(query, p=3) 호출
         ↓
  Edge Falloff 적용 → Z 높이 결정
         ↓
  Z값 기준 색상 결정 (ColorMapper)
         ↓
  Elevated Mesh + Flat Mesh + 범례
```

### 4.1 관측점: 법정동 중심점

URSUS의 관측점은 법정동 중심점이다. 각 중심점은 VWorld API로부터 수집한 법정동 경계 폴리라인의 기하 중심(centroid)이다. 관측값은 `BuildOverlayValues()`가 생성한 0~1 스칼라다.

서울 내 분석 범위 기준 약 430~470개의 관측점이 생성된다. 이 수가 IDW 계산의 N이다. 버텍스 하나를 계산할 때마다 이 N개를 전부 순회하므로, 버텍스 수 × N번의 연산이 발생한다.

### 4.2 메쉬 생성

`MeshingParameters.MaximumEdgeLength`(resolution 파라미터, 기본 100m)로 메쉬 밀도를 제어한다. 이 값이 작을수록 버텍스가 촘촘해지고 표현이 정밀해지지만, IDW 계산량도 선형으로 증가한다.

### 4.3 Edge Falloff

IDW만으로는 경계에서 값이 급격히 잘린 것처럼 보일 수 있다. URSUS는 각 버텍스에서 외곽 경계(unionBoundary)까지의 거리를 계산하고, 경계에 가까울수록 Z높이를 감쇠시킨다.

```
s = (dist / maxDist) ^ edgeFalloff
Z = IDW_normalized_height × heightScale × s
```

이 감쇠는 시각화에서 경계가 자연스럽게 "땅으로 내려앉는" 효과를 만들어 분석 영역의 가독성을 높인다. 단, 색상도 이 Z값 기준으로 결정되므로 경계 근방 버텍스는 내부 값이 아무리 높아도 낮은 색상을 가진다. 이 점은 색상이 "순수한 데이터 값"이 아니라 "데이터값 × 위치 감쇠"를 반영한다는 것을 의미하며, 시각적 해석 시 유의해야 한다.

## 5. 실무에서의 주 사용처

IDW는 1968년 Donald Shepard가 제안한 이후 수십 년간 지리정보시스템(GIS)의 기본 보간 도구로 활용되었다. 현재도 다양한 분야에서 쓰인다.

### 5.1 환경 모니터링
- **대기질 예측**: 측정소가 있는 위치의 PM2.5, 오존 농도를 기반으로 측정소 없는 지점의 공기 질 추정
- **강수량 분포**: 기상 관측소 데이터로 비가 오지 않은 지역의 강수량 추정
- **지하수 수위**: 시추공 관측 데이터에서 전체 대수층(aquifer) 수위 면 생성

### 5.2 지형 분석
- **DEM(Digital Elevation Model) 생성**: 드론이나 GPS로 취득한 산발적인 고도 측정치로 연속 지형면 생성
- **토양 특성 매핑**: 표본 지점의 토양 pH, 질소 함량을 공간 분포로 변환

### 5.3 도시 분석 (URSUS의 맥락)
- **부동산 가격 히트맵**: 실거래 지점의 가격을 공간 연속 면으로 표현
- **소득·소비 분포**: URSUS와 동일한 접근 — 행정구역 단위 데이터를 연속 공간으로 변환
- **유동인구 밀도**: 카드 결제, 통신사 위치 데이터 등의 집계 지점을 보간
- **교통 접근성**: 정류장·역 위치에서의 접근성 점수를 면으로 변환

### 5.4 의료 역학
- **질병 발생률 매핑**: 지역별 환자 수 데이터를 공간 보간해 위험 지역 식별
- **의료 접근성**: 병원·클리닉 위치를 기반으로 의료 서비스 접근성 면 생성

### 5.5 산업 분야
- **광물 탐사**: 시추 지점의 광물 함량을 보간해 채굴 가능 영역 추정 (광산공학에서 Kriging과 함께 표준 도구)
- **해양 수온·염도**: 관측 부이 데이터로 해양 물리 특성 분포 생성

## 6. IDW의 장점

### 6.1 직관적이고 구현이 단순하다
수식 자체가 명확하고, 파라미터가 p 하나다. 이해하기 쉽고 디버깅이 용이하다. Kriging처럼 반변동도(variogram)를 모델링하거나, RBF(Radial Basis Function)처럼 선형 시스템을 풀 필요가 없다.

### 6.2 결정론적(deterministic)이다
동일한 입력에 대해 항상 동일한 출력이 나온다. 확률적 요소가 없으므로 재현성이 보장되고, 결과 해석이 직관적이다.

### 6.3 정확 보간(exact interpolator)이다
관측점 위치에서는 반드시 해당 관측값을 반환한다. 즉, 알려진 데이터를 왜곡하지 않는다. 이는 데이터의 신뢰성이 높을 때 중요한 특성이다.

### 6.4 계산 복잡도가 선형이다
점 하나를 보간하는 데 O(N) 연산이 필요하다(N = 관측점 수). 공간 인덱스(k-d tree 등)를 쓰면 더 줄일 수 있다. 대규모 메쉬에도 합리적인 속도를 낸다.

### 6.5 지역적으로 적응적이다
p 값이 크면 사실상 가장 가까운 몇 개 관측점만 의미 있는 가중치를 가지므로, 관측점 분포가 불균일해도 국소 패턴을 잘 포착한다.

### 6.6 도메인 지식 없이 쓸 수 있다
Kriging은 현상의 공간 자기상관(autocorrelation) 구조를 사전에 파악해야 한다. IDW는 그런 사전 지식 없이도 적용 가능하다.

## 7. IDW의 한계

### 7.1 Bull's Eye 현상 (가장 근본적인 한계)

IDW의 가장 유명한 단점이다. 관측점 주변에 거리에 따라 급격히 값이 떨어지는 "원형 등고선" 패턴이 생기는 현상이다.

현실에서 공간 현상은 주변 환경의 맥락을 따른다. 하천을 끼고 있거나, 도로를 따라 선형으로 분포하거나, 산을 기준으로 급변할 수 있다. IDW는 이런 **물리적 구조**를 전혀 모르고, 그저 "가까울수록 닮는다"는 원칙만 적용한다. 결과적으로 현실과 괴리된 원형 패턴이 시각화에 나타난다.

URSUS에서 이 현상이 의미하는 바: 법정동 중심점은 다각형의 무게 중심이므로, 중심점 주변에서 인위적으로 높은(또는 낮은) 색상 영역이 생길 수 있다.

### 7.2 관측점 밀도에 민감하다

관측점이 밀집된 지역은 세밀한 변화가 잘 반영되고, 드문 지역은 넓은 면적이 가장 가까운 관측점 하나의 값으로 지배된다. 서울의 경우 도심(중구, 종로구)은 법정동이 많고 외곽(강서, 노원 등)은 비교적 적어 이 불균형이 결과에 영향을 준다.

### 7.3 관측값 범위를 벗어나지 않는다

IDW는 항상 관측값의 [min, max] 범위 안에서 결과를 반환한다. 이는 안전성의 장점이기도 하지만, 관측점들보다 더 극단적인 값이 존재하는 영역(예: 실제 최고가 지역이 관측점 사이에 있는 경우)을 표현할 수 없다는 한계이기도 하다. Kriging은 이 경우에 예측 구간(confidence interval)을 함께 제공해 불확실성을 표현할 수 있다.

### 7.4 이방성(Anisotropy)을 표현할 수 없다

기본 IDW는 유클리드 거리를 쓰므로, 방향에 따라 상관관계가 달라지는 현상을 표현하지 못한다. 예를 들어 "강을 따라 선형으로 분포하는 오염"이나 "능선을 기준으로 완전히 다른 기후"는 단순 거리 기반으로는 포착할 수 없다.

### 7.5 p 선택이 임의적이다

p가 결과에 결정적인 영향을 미치지만, 어떤 p가 "옳은지"를 데이터에서 자동으로 도출하는 방법이 없다. 실무에서는 교차검증(cross-validation, leave-one-out)으로 최적 p를 찾기도 하지만, URSUS처럼 시각화 목적으로 쓸 때는 시각적 판단에 의존하게 된다. URSUS 기본값 p=3은 p=2보다 국소성이 강해 법정동 단위 데이터의 패턴을 비교적 선명하게 드러내지만, "왜 3인가"에 대한 이론적 근거는 없다.

### 7.6 불확실성 추정이 불가하다

IDW는 각 예측값에 신뢰도 정보를 부여하지 못한다. 관측점이 밀집된 지역과 희박한 지역의 예측이 동일하게 "확정적인 값"으로 표현된다. Kriging은 각 예측값과 함께 분산(variance)을 반환해 "이 지역 예측은 신뢰도가 낮습니다"를 표현할 수 있다.

## 8. IDW vs 대안 보간법 비교

| 방법 | 복잡도 | 정확 보간 | 불확실성 | 이방성 | 사용 난이도 |
|---|---|---|---|---|---|
| **IDW** | O(N) | ✅ | ❌ | ❌ | 매우 쉬움 |
| Kriging (OK) | O(N²~N³) | ✅ | ✅ | ✅ | 어려움 |
| RBF | O(N²~N³) | ✅ | ❌ | 가능 | 중간 |
| Spline | O(N log N) | ✅ | ❌ | ❌ | 중간 |
| Natural Neighbor | O(N log N) | ✅ | ❌ | ❌ | 중간 |
| NN (최근접) | O(log N) | ✅ | ❌ | ❌ | 매우 쉬움 |

**Kriging**은 IDW의 이론적으로 더 엄밀한 대안이다. 공간 자기상관 구조(반변동도)를 명시적으로 모델링하기 때문에 예측 정확도가 높고 불확실성도 정량화할 수 있다. 그러나 변동도 피팅, 행렬 역산 등 추가 계산과 도메인 지식이 필요해 도시 분석 탐색 단계보다는 최종 분석 단계에 적합하다.

**Natural Neighbor(자연 이웃 보간)** 는 Bull's Eye 현상이 없고, 점 분포에 자연스럽게 적응하는 장점이 있어 IDW의 실용적 대안으로 주목받는다.

## 9. URSUS 맥락에서의 적절성 평가

### 적합한 이유
- 분석 목적이 **탐색적(exploratory)** 이다 — 절대값 정밀도보다 법정동 간 상대적 패턴 파악이 목표
- 관측점(법정동 중심점)이 서울 전역에 비교적 균등하게 분포한다
- 계산이 빠르고, 파라미터(p, resolution)를 GH 와이어로 실시간 조정할 수 있다
- 결과가 직관적으로 해석 가능하다

### 유의해야 할 점
- **색상이 데이터값이 아니라 Z값(IDW × edge falloff)을 반영**한다 — 경계 근방의 낮은 색상은 데이터가 낮아서가 아니라 edge falloff 때문일 수 있다
- **Bull's Eye** 가 생길 수 있다 — 특정 법정동 중심점이 지리적으로 고립되어 있으면 그 주변에서 인위적 패턴이 나타난다
- **p=3은 임의적 선택**이다 — 데이터 특성에 따라 p=2가 더 자연스러울 수 있다. GH 컴포넌트에서 실시간으로 바꿔가며 판단하는 것이 좋다
- **법정동 데이터를 연속 면으로 보간하는 것 자체가 일종의 추상화**다 — 법정동 경계가 갖는 행정적·물리적 의미를 희석시킨다. 이 시각화는 "공간적 경향성"을 보여주는 것이지 "법정동별 정확한 값 지도"가 아니다.

---

## 10. 결론

IDW는 "단순하지만 충분히 유용한" 알고리즘의 교과서적 예다. 수십 년 전에 제안된 수식이지만, 탐색적 도시 데이터 시각화에서 여전히 현역이다. URSUS가 IDW를 선택한 것은 — 수백 개 법정동의 이산 데이터를 연속 공간 면으로 변환해 직관적인 도시 패턴을 보여주는 목적에서 — 합리적인 선택이다.

다만 이 시각화를 해석할 때는 항상 "이것은 보간된 추정치이며, edge falloff에 의해 경계는 항상 낮게 표현된다"는 전제를 염두에 두어야 한다. 더 정밀한 분석이 필요한 단계에서는 Kriging이나 공간 통계 기법으로 전환하는 것이 적절하다.
