# URSUS Usability Improvement PRD

**Product:** URSUS (Urban Research with Spatial Utility System)
**Author:** TeacherChae
**Date:** 2026-03-31
**Status:** Draft
**Target:** v0.2.0

---

## 1. 현재 상태 요약

URSUS는 서울시 법정동 단위의 공간 데이터(소득, 인구, 대중교통 등)를 수집하고, IDW 보간을 통해 Grasshopper 위에서 3D 메시로 시각화하는 C# 플러그인이다.

현재 달성된 것:

- Python → C# .NET 클래스 라이브러리 마이그레이션 완료
- `.gha` 빌드 성공, Grasshopper 네이티브 플러그인으로 동작
- Solver / Visualizer 2개 컴포넌트 분리로 재연산 범위 최적화
- 4종 데이터셋 지원 (월평균 소득, 상주인구, 생활인구, 대중교통 승차)
- Weighted overlay 방식의 다중 데이터 합성
- 캐시 시스템 (TTL 30일)

핵심 성능 지표: 두 컴포넌트 합산 약 12초 (초회 API 호출 포함), 캐시 히트 시 2~3초.

## 2. 문제 정의

`docs/260320_todo_list.md`와 코드 분석에서 도출된 사용성 문제들:

### 2-1. 설치 과정이 번거롭다

현재 사용자가 플러그인을 쓰려면:
1. `.gha` + `.dll` 파일들을 `%AppData%\Grasshopper\Libraries\`에 수동 복사
2. 파일 속성에서 "차단 해제"를 하나씩 클릭
3. `adstrd_legald_mapping.json`을 어딘가에 배치
4. `appsettings.json`에 API 키를 수동 입력하거나 컴포넌트 입력에 직접 연결
5. Rhino 재시작

`URSUS.Setup.exe`가 DLL2GHA.md에 설계되어 있지만 아직 구현되지 않았다.

### 2-2. Solver 컴포넌트의 입력이 과다하다

현재 `URSUSSolverComponent`의 입력:
- VWorld Key (필수)
- Seoul Key (필수)
- DataSet (필수 — Value List로 연결)

**address1, address2는 이미 상수로 내재화되었다.** 하지만 여전히 남아있는 문제:
- API 키를 매번 Panel로 연결하거나 `appsettings.json`을 수동으로 만들어야 한다
- `adstrd_legald_mapping.json`이 DLL과 같은 폴더에 반드시 있어야 한다 — 사용자가 이 파일의 존재를 인지하고 배치해야 한다

### 2-3. 매핑 파일 배포 방식이 불명확하다

`KIKmix.xlsx`에서 변환한 `adstrd_legald_mapping.json`은:
- 현재 `refs/` 폴더에 있지만, 빌드 결과물에 자동 포함되지 않는다
- 사용자가 직접 이 파일을 DLL 옆에 놓아야 동작한다
- 이 파일 없이는 행정동↔법정동 코드 매핑이 불가하므로 전체 파이프라인이 실패한다

### 2-4. 결과 데이터의 활용 경로가 없다

Solver가 반환하는 법정동별 데이터(코드, 이름, 면적, 소득 등)가 GH Panel에만 뿌려진다. 사용자가 이 데이터를 보고서나 스프레드시트로 내보내려면 수동으로 복사해야 한다.

### 2-5. 분석 범위가 서울 고정이다

`VworldApiParser`에서 `fullNm.Contains("서울")` 필터가 하드코딩되어 있고, address1/address2도 상수다. 현재 기획상 서울만 다루지만, 향후 확장 가능성을 고려한 설계가 필요하다.

## 3. 목표

이 PRD가 다루는 범위는 **"플러그인을 처음 받은 사용자가 5분 안에 결과를 볼 수 있게 만드는 것"** 이다.

| 목표 | 측정 기준 |
|---|---|
| 설치 완료까지 3단계 이내 | Setup.exe 실행 → API 키 입력 → 설치 완료 |
| Solver 컴포넌트의 필수 외부 입력 = 0개 | API 키는 설정 파일에서 자동 로드, DataSet은 기본값 제공 |
| 매핑 파일 배포 자동화 | 사용자가 JSON 파일의 존재를 몰라도 동작 |
| 결과 데이터 내보내기 지원 | CSV 또는 Excel 파일로 한 번에 출력 |

## 5. 상세 설계

### F-01: Setup.exe 구현

`DLL2GHA.md`의 Phase 2 설계를 기반으로 구현한다.

**동작 흐름:**

```
URSUS.Setup.exe 실행
  │
  ├── 1. Grasshopper Libraries 경로 자동 감지
  │      %AppData%\Grasshopper\Libraries\
  │
  ├── 2. API 키 입력 (VWorld, 서울 열린데이터)
  │      - 둘 다 선택사항 (나중에 변경 가능)
  │      - 입력 시 appsettings.json에 저장
  │
  ├── 3. 파일 복사 + 차단 해제
  │      URSUS.GH.gha
  │      URSUS.dll
  │      Clipper2Lib.dll
  │      appsettings.json
  │
  └── 4. 완료 메시지 + "Rhino 재시작 필요" 안내
```

**배포 패키지 구조:**

```
URSUS-v0.2.0/
├── URSUS.Setup.exe        (self-contained, single file)
├── URSUS.GH.gha
├── URSUS.dll
└── Clipper2Lib.dll
```

**기술 결정:**
- WinForms (net7.0-windows), `PublishSingleFile` + `SelfContained`
- Zone.Identifier ADS 삭제로 Windows 차단 자동 해제
- 관리자 권한 불필요 (`%AppData%`는 사용자 폴더)

### F-02: 매핑 파일 Embedded Resource화

현재 `URSUSSolver`가 DLL 옆의 파일 시스템에서 `adstrd_legald_mapping.json`을 읽는다. 이것을 `URSUS.dll`에 내장한다.

**변경 사항:**

`URSUS.csproj`에 추가:
```xml
<ItemGroup>
  <EmbeddedResource Include="..\..\refs\adstrd_legald_mapping.json"
                    LogicalName="URSUS.adstrd_legald_mapping.json" />
</ItemGroup>
```

`MappingLoader.cs` 수정:
```csharp
public static Dictionary<string, List<string>> Load()
{
    // 외부 파일 없이 어셈블리 내장 리소스에서 로드
    var assembly = typeof(MappingLoader).Assembly;
    using var stream = assembly.GetManifestResourceStream(
        "URSUS.adstrd_legald_mapping.json")!;
    using var reader = new StreamReader(stream);
    string json = reader.ReadToEnd();
    return Parse(json);
}
```

`URSUSSolver` 생성자에서 `_mappingJsonPath` 파라미터 제거.

**효과:** 사용자가 JSON 파일의 존재 자체를 몰라도 플러그인이 동작한다.

### F-03: API 키 자동 로드 체계

API 키 탐색 우선순위:

```
1. GH 컴포넌트 입력 (VK, SK 포트에 직접 연결)
2. appsettings.json (URSUS.GH.gha와 같은 폴더)
3. 환경 변수 (URSUS_VWORLD_KEY, URSUS_SEOUL_KEY)
```

1번이 비어있으면 2번을 시도하고, 2번도 없으면 3번을 시도한다. 모두 실패하면 에러 메시지에 Setup.exe 실행을 안내한다.

**이미 구현된 부분:** `URSUSSolverComponent.SolveInstance()`에서 `LoadConfig()` 호출로 appsettings.json 읽기는 완료됨. 환경 변수 fallback만 추가하면 된다.

### F-04: DataSet 기본값

현재 `DataSet` 입력이 비어있으면 컴포넌트가 즉시 return한다. 입력이 없을 때 전체 데이터셋을 기본 선택하도록 변경한다.

```csharp
// URSUSSolverComponent.SolveInstance() 내부
var dataSet = new List<string>();
if (!DA.GetDataList(2, dataSet) || dataSet.Count == 0)
{
    dataSet = new List<string>
    {
        URSUSSolver.DS_AVG_INCOME,
        URSUSSolver.DS_RESIDENT_POP,
        URSUSSolver.DS_TRANSIT
    };
}
```

**효과:** DataSet 입력에 아무것도 연결하지 않아도 3종 데이터의 weighted overlay가 자동 실행된다. Value List를 연결하면 선택적으로 데이터셋을 줄일 수 있다.

### F-05: CSV 내보내기 컴포넌트

`URSUS > Export` 카테고리에 새 컴포넌트를 추가한다.

**컴포넌트: URSUS Exporter**

입력:
| 이름 | 타입 | 설명 |
|---|---|---|
| Legal Codes | List\<string\> | Solver 출력 연결 |
| Names | List\<string\> | Solver 출력 연결 |
| Areas | List\<double\> | Solver 출력 연결 |
| Values | List\<double\> | Solver 출력 연결 |
| FilePath | string | 저장 경로 (기본: 바탕화면/URSUS_export.csv) |
| Run | bool | Button 연결 |

출력:
| 이름 | 타입 | 설명 |
|---|---|---|
| Result | string | "저장 완료: C:\...\URSUS_export.csv (42건)" |

CSV 형식:
```
법정동코드,법정동명,면적(㎡),오버레이값
1111010100,서울특별시 종로구 청운동,485213.5,0.72
...
```

### F-06: 에러 메시지 개선

현재 에러 메시지 예시:
```
API 키가 설정되지 않았습니다.
```

개선:
```
API 키가 설정되지 않았습니다.
→ URSUS.Setup.exe를 실행하여 설정하거나, VK/SK 입력에 Panel을 연결하세요.
→ 도움말: https://github.com/TeacherChae/URSUS#setup
```

주요 에러 상황별 메시지 표준화:
| 상황 | 메시지 |
|---|---|
| API 키 미설정 | Setup.exe 실행 또는 직접 입력 안내 |
| API 응답 오류 | HTTP 상태 코드 + 재시도 안내 |
| 캐시 파손 | 캐시 파일 경로 + 삭제 후 재실행 안내 |
| 법정동 0건 수집 | 주소 범위 확인 안내 (현재는 서울 고정) |

### F-07: edgeFalloff 파라미터 내부화

Visualizer 컴포넌트의 입력이 현재 11개로 많다. `edgeFalloff`는 사용자가 직접 조정할 일이 거의 없으므로 내부 자동 계산으로 전환한다.

현재 IDWVisualizer 생성자에서 제거하고, `Build()` 내부에서 boundary bbox 크기 대비 비율로 자동 결정:
```csharp
double edgeFalloff = 2.0; // 내부 고정값
```

**효과:** Visualizer 입력이 11개 → 10개로 감소. 고급 사용자를 위해 향후 "Advanced" 컴포넌트 변형으로 노출 가능.

```

├── F-02  매핑 파일 Embedded Resource화
```

Phase 1 완료 시점에 **"API 키만 넣으면 바로 동작"** 상태가 된다.
Phase 2 완료 시점에 **"Setup.exe 한 번 실행하면 끝"** 상태가 된다.

## 7. 변경 영향도

| 파일 | 변경 내용 | 영향 |
|---|---|---|
| `URSUS.csproj` | EmbeddedResource 추가 | DLL 크기 ~60KB 증가 |
| `MappingLoader.cs` | 리소스 스트림 로드로 변경 | 외부 파일 의존성 제거 |
| `URSUSSolver.cs` | 생성자에서 mappingJsonPath 제거 | **Breaking change** — GH wrapper 동시 수정 필요 |
| `URSUSSolverComponent.cs` | 기본값 로직, 환경 변수 fallback | 기존 동작 호환 |
| `VisualizerComponent.cs` | edgeFalloff 입력 제거 | **Breaking change** — 기존 GH 파일에서 해당 와이어 끊김 |
| 신규: `ExporterComponent.cs` | CSV 내보내기 | 추가 컴포넌트 |
| 신규: `src/URSUS.Setup/` | 설치 도구 | 별도 프로젝트 |

## 8. 최종 사용자 경험 (After)

### 최초 설치

```
1. URSUS-v0.2.0.zip 다운로드 → 압축 해제
2. URSUS.Setup.exe 실행
3. API 키 입력 (선택) → "설치" 클릭
4. Rhino 재시작
```

### Grasshopper 사용

```
1. URSUS 탭 → Solver 컴포넌트 드래그
   (입력 없이 바로 실행 — API 키는 설정 파일에서, DataSet은 전체 기본)
2. URSUS 탭 → Visualizer 컴포넌트 드래그
   (Solver 출력을 연결하면 즉시 시각화)
3. colorStyle, resolution 슬라이더로 시각화 조정
   (API 재호출 없이 즉시 반영)
4. (선택) Exporter 컴포넌트로 CSV 내보내기
```

### Before vs After 비교

| 항목 | Before (v0.1) | After (v0.2) |
|---|---|---|
| 설치 단계 | 5단계 (수동 복사 + 차단 해제) | 3단계 (Setup.exe) |
| Solver 필수 외부 입력 | 1~3개 (API 키 + DataSet) | 0개 |
| 매핑 파일 관리 | 사용자가 직접 배치 | 자동 (DLL 내장) |
| 결과 내보내기 | 불가 (Panel 복사) | CSV 원클릭 |
| 에러 대응 | 영문 예외 메시지 | 한글 가이드 + 해결 링크 |
