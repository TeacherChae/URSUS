# 행정동별 생활인구

> **제품 상태: 비활성.** 현재 `DataSeoulApiParser`의 생활인구 API는 `NotSupportedException`을 반환한다. 아래 내용은 원천 데이터 조사 기록이며 현재 URSUS 결과 계약을 설명하지 않는다.

## 개요

서울시 행정동 단위의 일별·시간대별 생활인구 추정 데이터.
KT 통신 데이터 기반으로 추정된 값으로, 주민등록인구(상주인구)가 아닌
실제 해당 지역에 체류한 인구를 반영한다.

## 출처

- **기관**: 서울특별시 (서울 열린데이터 광장)
- **서비스명**: `SPOP_LOCAL_RESD_DONG`
- **API 엔드포인트**: `http://openapi.seoul.go.kr:8088/{키}/xml/SPOP_LOCAL_RESD_DONG/{start}/{end}/`
- **데이터 페이지**: https://data.seoul.go.kr (서울시 우리마을가게 상권분석서비스 생활인구)

## 업데이트 주기

- **일별 × 시간대별** (24시간, `TMZON_PD_SE`: `00`~`23`)
- 총 레코드 수: **763,276행** (2026년 3월 기준)
- 예상 구조: 서울 행정동(~424개) × 24시간 × 복수 날짜

## 공간 단위 및 커버리지

- **공간 단위**: 행정동 (`ADSTRD_CODE_SE`, 8자리)
  - ※ 소득 데이터의 `ADSTRD_CD`와 필드명이 다름 — 코드값 자체는 동일 체계
- **커버리지**: 서울특별시 전체 행정동

## 주요 필드

| 필드명 | 타입 | 설명 |
|---|---|---|
| `STDR_DE_ID` | string | 기준일자 (YYYYMMDD, e.g. `20260315`) |
| `TMZON_PD_SE` | string | 시간대 구분 (`00`~`23`시) |
| `ADSTRD_CODE_SE` | string | 행정동코드 (8자리) — 집계 키 |
| `TOT_LVPOP_CO` | float | **총 생활인구수** — 사용 필드 |
| `MALE_F{연령대}_LVPOP_CO` | float | 남성 연령대별 생활인구 (0~9, 10~14, ... 70~74세) |
| `FEMALE_F{연령대}_LVPOP_CO` | float | 여성 연령대별 생활인구 |

연령대 구분: `0T9`, `10T14`, `15T19`, `20T24`, `25T29`, `30T34`, `35T39`,
`40T44`, `45T49`, `50T54`, `55T59`, `60T64`, `65T69`, `70T74`

## 과거 검토한 가공 방법

```
원천 후보: 행정동 × 날짜 × 시간대 레코드
  → 분석 목적에 맞는 기간·시간대 선택 필요
  → complete 조건과 행정동→법정동 의미 결정 필요
  → 성능/메모리 fixture 검증 후에만 제품 source로 활성화
```

- 전체 시간대 단순 평균을 제품 기본값으로 채택하지 않았다.
- 주간 활동, 야간 체류, 평일/주말은 의미가 다르므로 먼저 recipe와 observation policy가 필요하다.

## 알려진 한계

- KT 통신 데이터 기반 추정치 — 타 통신사 가입자 보정 방법론 비공개
- 야간 시간대(00~06시) 포함 평균이므로 상주인구 성격이 섞임
  (주간 활동 인구만 원한다면 `TMZON_PD_SE` 09~18 필터링 권장)
- 총 레코드가 많아(76만행+) 활성화 전 streaming, 기간 제한, page 상한과 peak memory 검증이 필요
- 행정동 경계 변경 시 과거 데이터와 코드 불일치 가능
